# BgRemover AI - Техническая Документация

## Описание проекта
**BgRemover AI** — это веб-приложение для автоматического удаления фона с изображений и создания токенов для виртуальных настольных игр (VTT). Главная особенность: вся обработка происходит **локально в браузере** с использованием ONNX Runtime и WebAssembly. Никакие данные не отправляются на удаленные серверы, что гарантирует безопасность и высокую скорость работы после первичной загрузки моделей.

## Архитектура
Приложение состоит из клиентской части (HTML/CSS/JS) и простого локального сервера (Python / Node.js / PowerShell), необходимого исключительно для настройки CORS-заголовков (`Cross-Origin-Opener-Policy` и `Cross-Origin-Embedder-Policy`). Эти заголовки требуются браузерам для активации `SharedArrayBuffer` и многопоточности WebAssembly.

### Основные файлы
1. **`index.html`** - Структура интерфейса, разделенная на экраны (`#upload-screen`, `#studio-screen`, `#bulk-screen`, `#token-screen`).
2. **`app.js`** - Главный модуль логики приложения:
   - Инициализация и вызов функции `removeBackground` из пакета `@imgly/background-removal`.
   - Управление состояниями экранов (Одиночный, Пакетный, Токены).
   - Canvas-логика для отрисовки и трансформации Токенов.
   - Скачивание файлов (одиночное или упаковка в ZIP через `JSZip`).
3. **`style.css`** - Стили приложения (переменные, темы, отзывчивость).
4. **`start.py` / `start.ps1` / `run.bat`** - Скрипты локального запуска HTTP-сервера с поддержкой заголовков изоляции.

## Режимы работы

### 1. Студия редактирования (Одиночный режим)
После удаления фона объект можно трансформировать (масштаб, поворот, перемещение) поверх различных фонов (прозрачный, цвет, градиент, фото).
**Слои (DOM)**:
- `custom-bg-layer`: Фон (цвет/изображение).
- `workspace-layer`: Слой с перетаскиваемым объектом (`cutout-draggable`).
- `layer-original` / `layer-processed`: Используются для интерактивного ползунка-шторки "До/После" (отображаются только во вкладке "Прозрачный фон").

### 2. Пакетная обработка
Управление очередью загруженных изображений `bulkQueue`. Обработка файлов идет последовательно, чтобы избежать перегрузки памяти браузера (WASM-ограничения).

### 3. Токен-Редактор
Рисует VTT токены на HTML5 `<canvas>`.  
*Функция отрисовки:* `drawToken(canvas, width, height, isExport)`
Она рендерит токен по слоям в следующем порядке:
1. **Подготовка слоя персонажа (Временный холст `tempCanvas`)**:
   Персонаж масштабируется, позиционируется и поворачивается на отдельном буферном холсте. На этом же этапе к нему применяется маска ластика `tokenMaskCanvas` с использованием `globalCompositeOperation = 'destination-out'`. Это изолирует область стирания и предотвращает стирание рамки или фона токена.
2. **Отрисовка фона внутри круга**: Заливка фона токена (Цвет, Градиент или изображение).
3. **Персонаж (внутри круга)**: Отрисованный на временном холсте персонаж проецируется на основной холст с обрезкой по кругу радиуса `r` (через `ctx.clip()`).
4. **Рамка кольца**: Векторная отрисовка выбранного стиля (Золото, Серебро и др.) или отрисовка художественных текстур (Руны, Лёд, Пламя, Дракон), загружаемых из PNG-ассетов.
5. **3D Pop-out (Выход за рамку)**: Слой персонажа (с временного холста) рисуется повторно с обрезкой strictly по верхней прямоугольной области (без вырезания круга). Это позволяет персонажу перекрывать рамку в верхней части, скрывая её контур и создавая реалистичный 3D-эффект.

## Как добавить новый функционал (Гайд для ИИ)

### Добавление новой векторной рамки для токена:
1. В `index.html` добавить `option` в `<select id="token-ring-select">`.
2. В `app.js` найти `if (tokenRingType === '...'){` внутри функции `drawToken`. Добавить блок `else if` и нарисовать пути/градиенты с помощью `ctx`.

### Добавление новых статусов-эффектов:
1. Добавить `option` в `#token-status-overlay`.
2. В функции `drawToken` в секции `// 6. Draw Status Effects` добавить логику процедурной генерации эффекта внутри обрезки (`ctx.clip()`).

### Отладка
Если `SharedArrayBuffer` не определен в консоли, проверьте, запущен ли сервер через `start.py`, так как прямое открытие `index.html` в браузере (через `file://`) не поддерживает многопоточность ONNX.
